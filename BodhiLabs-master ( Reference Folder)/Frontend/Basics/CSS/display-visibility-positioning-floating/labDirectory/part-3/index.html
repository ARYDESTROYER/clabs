The Art of Web Layout

In the early days of web design, creating layouts was a significant challenge. Designers often used tables to structure their pages, a method that was functional but not semantically correct. The introduction of CSS brought new tools, and one of the most important for layout was the 'float' property.

The float property was originally designed to allow text to wrap around images, much like you see in a newspaper or magazine. By setting 'float: left' on an image, the image would move to the left edge of its container, and any subsequent text would flow neatly around its right side. This was a revolutionary step for web typography and design.

"Float was the cornerstone of CSS layouts for many years."

Designers soon realized that they could float entire divs, not just images. By floating one div to the left and another to the right, they could create multi-column layouts. This became the standard technique for building complex web pages for a long time. However, this method had its own set of challenges, such as clearing floats to prevent layout issues and dealing with containers that would collapse in height.

When you float an element, it is taken out of the normal document flow. This can cause the parent container to not recognize its height. The solution was to use the 'clear' property on an element that follows the floated items to restore the normal document flow.
